# What's New

## New Features

### Context-Based Volume Control

- Added support for **separate volume levels** for frontend and in-game music playback.
- Volume automatically switches based on the current game context (Frontend vs. In-game).
- New overlay interface displays the **active context's volume slider first**, with the secondary context below.
- Settings are saved to `ecm-r.x86.ini` for persistent configuration across sessions.

### Configuration Changes

#### Updated `[core]` Section

The generated `ecm-r.x86.ini` now includes:

```ini
[core]
playlist = "Music"
volume = "100"
frontend_volume = "100"
ingame_volume = "100"
version = "..."
```

**New settings:**
- `frontend_volume`: Volume level for menu/frontend playback. Range: 0-100. Default: 100
- `ingame_volume`: Volume level for racing/in-game playback. Range: 0-100. Default: 100
- `volume`: Legacy fallback volume (retained for backwards compatibility). Range: 0-100. Default: 100

**Behavior:**
- When playing in the frontend, `frontend_volume` is applied.
- When playing during a race, `ingame_volume` is applied.
- If context-specific volumes are unavailable, the legacy `volume` is used as fallback.

### Backward Compatibility

- Existing configurations automatically gain the new volume settings on next launch.
- The legacy `volume` setting is preserved and acts as a fallback.
- If `frontend_volume` or `ingame_volume` are missing from an existing config, they are populated with the value from `volume`.
- All previous features remain unchanged.

## Technical Changes

### BASS API Wrapper

- Added `BASS_ChannelSetAttribute` wrapper to the BASS API module.
- Implemented `channel_set_attribute()` and `set_channel_volume()` helper functions.
- Volume is applied per-channel using the `BASS_ATTRIB_VOL` attribute instead of global stream volume.

### Volume Management

- Added `current_context_volume()` function that returns the appropriate volume based on game state.
- Added `apply_current_context_volume()` function that applies the context-aware volume to the active playback channel.
- Volume is reapplied when:
  - A new song starts playing.
  - The game context changes (frontend to in-game or vice versa).
  - The user adjusts volume from the overlay.

### Overlay Integration

- **Context-Aware Display**: Volume slider labels change based on active context:
  - In Frontend: "Current Volume (Frontend)" appears first.
  - In-Game: "Current Volume (In-game)" appears first.
- Both sliders are always available, making it easy to adjust either context without restarting.
- Volume changes are saved immediately to `ecm-r.x86.ini`.

## Documentation Updates

- Updated `README.md`:
  - Added `frontend_volume` and `ingame_volume` to the default configuration example.
  - Documented both volume settings in the Core Settings section.
  - Updated feature description to mention context-based volume control.

## How to Use

### Default Behavior

1. No action required. Both frontend and in-game volumes are set to 100 by default.

### Adjust Volume from the Overlay

1. Press `F11` (default) to open the ECM-R overlay.
2. Navigate to **Actions**.
3. Adjust the **Current Volume** slider for the active context.
   - The slider label indicates which context's volume you're adjusting.
4. Optionally adjust the secondary context's volume below.

### Adjust Volume from Configuration File

1. Open `ecm-r.x86.ini`.
2. Edit `frontend_volume` and `ingame_volume` values under `[core]`.
3. Restart the game.

### Example Configurations

**Quiet frontend, loud racing:**
```ini
[core]
frontend_volume = "60"
ingame_volume = "100"
```

**Loud frontend, quiet racing:**
```ini
[core]
frontend_volume = "100"
ingame_volume = "70"
```

## Notes

- Volume adjustments are applied in real-time when using the overlay.
- The volume range (0-100) is mapped to BASS's internal range (0.0-1.0).
- Volume is clamped to ensure values stay within the valid 0-100 range.
- Switching between frontend and in-game automatically applies the correct volume without user interaction.
