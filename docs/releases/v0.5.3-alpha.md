# What's New

## New Features

### Runtime BASS Loading

- Removed build-time BASS SDK dependencies from the project.
- ECM-R now loads `bass.dll` dynamically at runtime using `LoadLibrary` and `GetProcAddress`.
- Added an internal `bass_api` wrapper so audio and player code use the runtime-loaded BASS entry points.

### Deployment Changes

- `bass.dll` is now an external runtime dependency and must be placed next to the ECM-R runtime files.
- The expected deployment keeps `ecm-r` runtime files and `bass.dll` together inside the `scripts` folder.

## Technical Changes

### Build and Runtime Integration

- Removed compile-time references to `bass.h`, `bass.lib`, and repository-managed BASS SDK files.
- Updated build scripts and project integration to rely on the official runtime DLL only.
- Kept BASS as an official externally downloaded dependency instead of bundling SDK artifacts in the repository.

## Documentation Updates

- Updated `CHANGELOG.md` for `v0.5.3-alpha`.
- Updated `README.md` to document the runtime `bass.dll` requirement.
- Updated `BUILDING.md` to clarify that users must download the official `bass.dll` separately.

## How to Use

1. Download the official `bass.dll` runtime from https://www.un4seen.com/.
2. Place `bass.dll` in the same `scripts` folder as the ECM-R runtime files.
3. Make sure your ECM-R files are deployed together, for example:
   - `scripts/ecm-r.x86.asi`
   - `scripts/ecm-r.x86.ini`
   - `scripts/bass.dll`
4. Launch the game normally.

## Notes

- ECM-R no longer requires bundled BASS SDK headers or libraries in the repository.
- Runtime playback depends on the presence of a compatible official `bass.dll`.
- The current runtime integration has been tested with BASS `v2.4.18.11`.
